在阿里云官网找到docker配置文档 Docker CE 镜像源站

配置镜像加速器

查看docker版本 docker version

语句分类

DDL

create 创建数据库或表 引擎engine=innodb 支持事务

ENGINE=InnoDB DEFAULT CHARSET=utf8

alter 修改数据库或表

drop 删除数据库或表

DML

insert into 插入

update set 更新

delete 删除

DQL

select from

DCL

对表中的用户和权限操作

TCL

事务管理 开启事务 提交事务 回滚事务

约束

primary key主键约束 不重复必须有值

unique 唯一约束

not null 非空约束

default 默认值

外键约束 foreign key(字段) reference 另一个表(字段)

检查约束 check(条件) 多条件 and or (mysql5检查约束没用，mysql8可以用)

查询语句

select中可以包括数学运算 select *, s * 12 from 表名

包括函数 select * , ifnull(列名，0) from 表名 为null的值转为0

别名 select ifnull(列名，0) as 别名 from表名

去除重复行 select distinct(列名) from 表名

描述表 desc 表名

where过滤行 select * from 表名 where 列名 >= ‘name’；

where限定行 select * from 表名 where 列名 between 10 and 100；日期加单引号 ‘1980-1-2’

select * from 表名 where 列名 in(a,b,c);

like 模糊查询 倒数第三个字符是s的 select * from 表名 where 列名 like ‘%s__’; 包含M like ‘%M%’

select * from 表名 where ac>2000 and (job = ‘a’ or job = ‘b’);

order by 排序 asc升序 desc降序 select * from 表名 order by 列名 asc/desc

限制记录的行数 limit 0,2 从0开始取2个 select * from 表名 order by 列名 limit 0,6;

多表连接 select * from customer c，ord d，item i where c.custid = o.custid and o.orderid = i.orderid 多表连接时，值为null可能造成记录丢失

非等值连接 薪水区间 select * from e ，s where e.sal between s.local and hisal n个表至少有n-1个关联关系

自连接 多表中使用自身的信息连接 select * from e1，e2 where e1.mgr = e2.empno

外连接 select * from emp e join dept t

左外连接 左边显示完整 select * from emp e left outer join dept t on(e.deptno = d.deptno) 右外连接 right outer join

通过union实现全外连接

分组查询 函数 select max(列名) from 表名 ； min() sum() avg() count() distinct去重

group by 分组 多列分组

having 筛选分组后的结果

子查询 where后 having后 from后 套整条语句

相关子查询 起别名 select * emp e where e.sal > (select avg(sal) from emp where depnto = e.deptno)

单行子查询 一行一列

多行子查询 多行一列 in any all

多列子查询 多列

from中使用子查询 把子查询的结果当作虚拟表使用

not in子查询中有空值 主查询全为空 使用 is not null

复制表(表结构) create table 新表名 like 表名 复制1表结构

子查询复制表(可以带数据) create table 新表名 子查询语句

重命名 rename table 表名 to 新表名 或 alter table 表名 rename 新表名

数据插入 insert into 表名(列名) values(数据)

日期时间 SELECT SYSDATE()
SELECT NOW()
SELECT CURDATE()

自增主键 id int primary key auto_increment 主键自动增长，插数据时写null 删除数据继续添加 编号不会连续

批量插入数据 insert into values中用，隔开 或 子查询插入

数据更新 update 表名 set 列名 = data_add(date,interval 10 day) 更新多个列用，隔开

更新日期向后10天 SELECT DATE_ADD(now(),INTERVAL 10 DAY)

删除 delete 不启用自动提交，删除后可以回滚，启用自动提交，不能回滚

索引

提高查询性能，本质是不断缩小查询范围

唯一索引 只要创建了主键或unique约束就有唯一索引

一般索引 create index 索引名 on 表名(列名)

全文索引（适合英文文章） create fulltext index 索引名 on 表名(列名)

组合索引 create index 索引名 on 表名(列名，列名)

索引失效 1.使用 != <> 2.用like并且第一个字符是%或者_ 3.数据量小 4.违反左匹配原则

缺点：增加表空间 降低了增删改的性能

索引的数据结构

B+ Tree索引 叶子节点存放key和data 其他内节点只存放key，叶子结点有一条引用链指向相邻叶子节点，具备更少的io次数，更稳定的查询效率，更适于范围查找的优势

Hash索引 （不适合范围查找）不支持顺序和范围查找

索引类型

聚簇索引 innodb

优点：查询速度非常快，对排序查找和范围查找优化

缺点：依赖有序的数据，更新代价大

非聚簇索引 MyIsam

优点：更新代价小

缺点：依赖于有序的数据，可能会二次查询

事务

原子性(要么成功要么失败) 无论该事务有多少条语句组成，都认为是一件事，不可拆分，要成功就全部成功，要失败就全部失败，中途失败，回退到成功之前的状态

一致性(不管成功失败数据保持一致) 事务只能将数据库状态从一个一致性状态转变为另一个一致性状态（一致性状态的含义是数据库中的数据应满足完整性约束）

隔离性(事务之间互不影响保证数据的正确性) 多个事务并发执行时，一个事务的执行不应该影响其他事务的执行（多个事务并发执行的结果与这些事务串行执行的结果一致）

持久性(一旦提交就会改变不能回滚)

数据库锁 在事务处理中可能出现锁的情况，等待一边处理完，另一边才能处理

insert update delete 默认存在事务

事务隔离级别

读未提交 允许读取尚未提交的数据变更，可能导致脏读，不可重复读，幻读

读已提交 允许读取并发事务已提交的数据 阻止脏读，可能出现幻读，不可重复读

可重复读 阻止脏读和不可重复读 可能产生幻读

可串行化 最高隔离级别 所有事务依次逐个执行 防止脏读，幻读，不可重复读
