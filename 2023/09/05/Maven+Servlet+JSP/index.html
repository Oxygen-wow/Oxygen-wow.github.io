Maven、Servlet、JSP

发表于2023-09-05|javaweb
|字数总计:1.1k|阅读时长:4分钟|阅读量:|评论数:

Maven

项目构建和jar包管理工具

lombok插件

自动生成setter，getter，toString，equals，构造

@Data
@NoArgsConstructor
@AllArgsConstructor

Servlet

servlet配置地址

自定义文件使用注解

导入jar包使用xml文件

请求同时附带数据，请求地址?数据名=数据值&数据名=数据值 接收时用request.getParameterValues();

Servlet之间无法传值，可以使用作用域达到传值的目的，调用setAttribute();存值，调用getAttribute()取值

pageContext 当前页面中存，当前页面取，servlet无法获取到pageContext

request 当前页面存，当前页面和转发页面取

session 当前页面存，当前会话取，用于保存登录用户 servlet获取 HttpSession session = request.getSession();

application 服务器不重启，数据不失效，所有用户共享 servlet获取 ServletContext appliction = request.getServletContext();

JSP

一种特殊的Servlet，注重页面内容输出

<% %>方式嵌入java代码 <%=变量名 %>输出变量值

转发 只发一次请求，地址栏不变，不能跨域

重定向 两次请求，地址栏改变，可以跨域

jsp不允许被浏览器直接访问，要经过servlet转发，将jsp放进WEB-INF目录中让浏览器访问不到

EL表达式

快速取出作用域中的数据 ${对象.名字} （其实调用对象的get方法，名字为去掉get首字母变小写，与属性名一样，如果第二个字母为大写，则第一个字母不用变）

不同作用域取相同key的值，取优先级高的 pageContext>request>session>appliction

JSTL

标签库，实现条件判断循环等业务逻辑

Servlet生命周期

加载类

实例化 （默认在Servlet被访问的时候创建，只会创建一次）

初始化 init方法（实例化的时候调用，只调用一次）

service方法

destory方法

LoadonStartUp 该配置代表让Servlet在服务器启动时创建对象，属性值大于0 @WebServlet(values=”/path” LoadonStartUp=1) 数小的先执行

如果有一段代码我们想在服务器启动的时候就运行，就可以把该代码放进一个Servlet的init方法中，并且配置该Servlet中的LoadOnStartUp

表单请求

form标签中action为请求地址，method为请求方式，按钮type为submit，每一个输入框都要有name属性

method属性

Get不会额外携带数据，传递的数据会放在地址栏，速度快，安全性差

Post 请求时额外携带数据，保存在FormData对象中发送到后端，也支持地址栏用问号传递数据，速度不如get，安全性好，涉及到文件上传，只能用post

获取数据用get提交数据用post

表单文件上传

form属性enctype=”multipart/form-data”

处理该请求的Servlet必须加注解@MultipartConfig

接收文件用request.getPart()

过滤器

一种组件，从发送请求到找到Servlet之间起到一个拦截作用

让类成为组件的两种方式：继承和实现

实现javax.servlet.Filter 配置@WebFilter注解或web.xml 实现抽象方法

小结

如果没有重写service方法就必须重写doGet方法和doPost方法

Controller层 用于处理Servlet 调用Service的实现类

Service层 用于逻辑处理 调用Dao层实现的数据库操作

Dao层 用于数据库操作

监听器

监听作用域生命周期，监听数据

实现接口，加@WebLsitener

request生命周期 ServletRequestListener
request作用域 ServletRequestAttributeListener
session生命周期 HttpSessionListener
session作用域 HttpSessionAttributeListener
ServletContext生命周期 ServletContextListener 启动时运行代码
